# Tríade OS · contrato inicial

Tríade OS es un plano de control cognitivo ejecutado sobre el sistema operativo
anfitrión. En esta fase no es un kernel, una distribución Linux ni un reemplazo
del sistema operativo. Coordina modelos, memoria, workers, nodos, permisos,
evidencia y ciclos auditables.

## Capas

1. Cognición: Central, Hipotálamo, Cristal y Qualia.
2. Memoria: Bodega Global, memoria episódica, semántica y aprendizaje candidato.
3. Ejecución: Runner, workers, Ollama y comandos preautorizados.
4. Borde global: señales locales y nodos federados autorizados con procedencia.
5. Gobierno: Safety, Permission Governor, integridad, regresión y rollback.
6. Refutación: ninguna afirmación arquitectónica se acepta como prueba de sí misma.

## Ontología teórica

- Central: teoría, razonamiento y estructura.
- Hipotálamo: emoción, intención y tonalidad.
- Bodega: memoria y continuidad.
- Qualia: alma simbólica y relacional; no demuestra conciencia subjetiva.
- Cristal: forma, coherencia y regulación morfológica.

## Invariantes

- `identity_core` es inmutable desde procesos autónomos.
- La entrada federada es evidencia, no verdad.
- El aprendizaje web entra como candidato, no como memoria estable.
- No existe shell libre en `full_local_guarded`.
- Los efectos externos requieren permiso y trazabilidad.
- `triade-omega` es un modelo derivado de Qwen hasta que exista entrenamiento y
  evaluación propios suficientes para describirlo de otro modo.

## Flujo global

```text
local_edge_signal + federated_nodes
                  ↓
federated_global_edge_context
                  ↓
Bodega Global → Hipotálamo → Central → Safety/Cristal → salida
                  ↓
       evidencia → refutación → aprendizaje candidato
```
